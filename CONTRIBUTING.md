## Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/veon_replit.git`
3. Follow the setup instructions in README.md
4. Create a new branch: `git checkout -b feature/your-feature-name`

## Code Style

### Backend (JavaScript/Node.js)
- Use ES6+ features
- Follow async/await patterns
- Add error handling for all async operations
- Use meaningful variable and function names
- Add comments for complex logic

### Frontend (React)
- Use functional components with hooks
- Follow React best practices
- Use Tailwind utility classes for styling
- Keep components focused and reusable
- Use meaningful component and prop names

## Commit Guidelines

Follow conventional commit format:

- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation changes
- `style:` Code style changes (formatting, etc.)
- `refactor:` Code refactoring
- `test:` Adding or updating tests
- `chore:` Maintenance tasks

Example: `feat: add voice input support for chat interface`

## Pull Request Process

1. Update documentation if needed
2. Test your changes thoroughly
3. Ensure the build passes: `npm run build`
4. Update CHANGELOG.md if applicable
5. Create a pull request with a clear description
6. Link any related issues

## Feature Requests

When suggesting a feature:
1. Check if it already exists or is planned
2. Clearly describe the feature and its benefits
3. Consider implementation complexity
4. Think about edge cases

## Bug Reports

When reporting a bug:
1. Use the bug report template
2. Include steps to reproduce
3. Provide expected vs actual behavior
4. Include screenshots if applicable
5. List your environment (OS, browser, Node version)

## Code Review

All contributions will be reviewed for:
- Code quality and style
- Test coverage
- Documentation
- Performance impact
- Security considerations
